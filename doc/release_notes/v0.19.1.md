# Release Notes v0.19.1

## Highlights

This patch release enhances `ds_target_update_service.sh` with a template
engine, listener port update support, root target handling, and optional
automatic derivation of service name and port from OCI PDB connection strings.

- **Enhanced**: `ds_target_update_service.sh` — configurable service name templates, port update, root target handling, `--from-oci` mode
- **New**: `oci_lookup_pdb_connection()` helper in `lib/oci_helpers.sh`

---

## Enhanced — `ds_target_update_service.sh`

### Template-based service naming

Service names are now derived from the target display name via configurable
brace-style templates rather than a hardcoded `_exa` derivation.

| Placeholder               | Expands to                               |
|---------------------------|------------------------------------------|
| `{pdb}` / `{PDB}`         | PDB name from display name (lower/upper) |
| `{cdb}` / `{CDB}`         | CDB name from display name (lower/upper) |
| `{cluster}` / `{CLUSTER}` | Cluster name (lower/upper)               |
| `{sid}` / `{SID}`         | Alias for cdb — useful in root templates |
| `{domain}`                | `DB_DOMAIN` value                        |

```bash
# Default template (backward-compatible result)
ds_target_update_service.sh -c prod --apply

# Custom template
ds_target_update_service.sh -c prod \
    --service-template "{PDB}_PAAS.{domain}" --apply

# PDB + root templates
ds_target_update_service.sh -c prod \
    --service-template "{pdb}_exa.{domain}" \
    --root-service-template "{cdb}.{domain}" --apply
```

Root targets (CDB$ROOT) require `--root-service-template` or are skipped
with a warning. Configure in `datasafe.conf`:

```bash
SERVICE_TEMPLATE="{pdb}_exa.{domain}"
ROOT_SERVICE_TEMPLATE="{cdb}.{domain}"
```

### Listener port update

```bash
# Explicit port (enables port update)
ds_target_update_service.sh -c prod --port 1565 --apply

# Use LISTENER_PORT from config
ds_target_update_service.sh -c prod --update-port --apply

# Port only (skip service update)
ds_target_update_service.sh -c prod --port 1522 --no-service-update --apply
```

### OCI PDB connection string lookup (`--from-oci`)

Automatically derives service name **and** port from the OCI PDB object's
`pdb-default` connection string (e.g. `host:1565/pdb01_PAAS.example.com`):

```bash
# Derive service+port from OCI PDB metadata
ds_target_update_service.sh -c prod --from-oci --apply

# PDB lookup in a separate compartment
ds_target_update_service.sh -c datasafe-comp \
    --pdb-compartment exacs-comp --from-oci --apply
```

Uses the new `oci_lookup_pdb_connection()` library function. Targets whose
PDB cannot be found in OCI are skipped with a warning.

### New library function — `oci_lookup_pdb_connection()`

```bash
oci_lookup_pdb_connection PDB_NAME COMPARTMENT_OCID
# Returns: "<service>|<port>" or exits 1 if not found
# Source:  lib/oci_helpers.sh (after ds_filter_targets_by_tags)
```

Queries `oci db pluggable-database list`, filters TERMINATED PDBs, and
parses the `pdb-default` connection string into service name and port.

## Testing

```bash
make test

# Default template, dry-run
ds_target_update_service.sh -c prod --dry-run

# Custom template
ds_target_update_service.sh -c prod \
    --service-template "{PDB}_PAAS.{domain}" --domain example.com --dry-run

# Port only (no service update)
ds_target_update_service.sh -c prod --port 1522 --no-service-update --dry-run

# Root + PDB templates, apply
ds_target_update_service.sh -c prod \
    --service-template "{pdb}_exa.{domain}" \
    --root-service-template "{cdb}.{domain}" \
    --update-port --apply

# Derive service+port from OCI PDB metadata
ds_target_update_service.sh -c datasafe-comp \
    --pdb-compartment exacs-comp --from-oci --apply
```
