# OraDBA Data Safe Extension v0.14.1

Release date: 2026-02-19

## Highlights

- Fixed embedded payload extraction to be robust against formatter-added whitespace.
- Ensured payload rebuild tooling handles marker formatting variations consistently.

## What Changed

### Fixed

`bin/ds_database_prereqs.sh` embedded payload extraction is now tolerant of:

- leading/trailing whitespace around `__PAYLOAD_BEGINS__` / `__PAYLOAD_END__`
- heredoc wrapper variants such as:
  - `: <<'__PAYLOAD_END__'`
  - `: << '__PAYLOAD_END__'`
- CRLF and blank lines in the payload area

This prevents payload decode/extract failures in `--embedded` mode after
formatting runs.

### Tooling

`scripts/update_embedded_payload.sh` now uses whitespace-tolerant marker matching
when replacing payload blocks, reducing formatter-related breakage in regenerated
payload sections.

The payload updater now also emits the `shfmt`-normalized heredoc wrapper style:

- `: << '__PAYLOAD_END__'`

This avoids recurring wrapper-only diffs in `bin/ds_database_prereqs.sh` when
running `make pre-commit`.

## Affected Files

- `bin/ds_database_prereqs.sh`
- `scripts/update_embedded_payload.sh`
