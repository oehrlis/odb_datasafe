# Release Notes - v0.23.0

**Release Date:** 2026-06-28
**Type:** Minor (M3 - Installer Hardening)

## Summary

M3 hardens `bin/install_datasafe_service.sh`: adds ERR/EXIT traps, routes all
log messages to stderr, resolves systemd binary paths at runtime, fixes the
`--install`/`--prepare` contract in dry-run mode, adds a hard validation error
for missing `oradba_dsctl.sh`, and reports the connector log directory check in
the dry-run plan output. All six REG-001..REG-006 regression tests are now green.

## Changes

### Bash Robustness

- `bin/install_datasafe_service.sh`: added minimal inline ERR and EXIT traps.
  Any unhandled error now prints the failing line and command before exiting,
  rather than silently leaving a partial installation (BASH-016).
- `bin/install_datasafe_service.sh`: `print_message` now routes ALL log levels
  (`SUCCESS`, `WARNING`, `INFO`, `STEP`) to stderr. Only undecorated structured
  output (file paths, plan lines) goes to stdout. Consistent with the rest of
  the framework (BASH-006).

### Installer Contract Fix (ARCH-008)

- `bin/install_datasafe_service.sh`: When `--install` detects a `User=` mismatch
  and auto-regenerates service files, the subsequent `chown` is now guarded by
  `$DRY_RUN`. In dry-run mode the chown is skipped (avoids spurious failure when
  the named user does not exist in the test environment).

### Path Resolution (ARCH-007 partial)

- `bin/install_datasafe_service.sh` `generate_sudoers_file()`: `systemctl` and
  `journalctl` paths are now resolved at generation time via `command -v` with
  conventional-path fallbacks. Both `/bin/systemctl` and `/usr/bin/systemctl`
  variants are included in the generated sudoers rule, covering distros that
  install systemd under `/usr/bin/` (Debian/Ubuntu) and those that use `/bin/`.

### Pre-install Validation (DEP-007)

- `bin/install_datasafe_service.sh` `prepare_service()`: when a registry alias
  is resolved (oradba_dsctl.sh mode), the dsctl binary is validated immediately.
  A missing or non-executable `oradba_dsctl.sh` now aborts `--prepare` with a
  clear error and `ORADBA_BASE` guidance, instead of producing a service file
  with a broken `ExecStart`.

### Log Directory Reporting

- `bin/install_datasafe_service.sh`: The dry-run plan output now includes a
  `Creating connector log directory` message when the log directory is absent.
  Previously this check only ran in the real install path (after the dry-run
  early return), making it invisible to operators previewing the installation.

### Regression Tests

- `tests/install_datasafe_service.bats`: Added REG-001..REG-006 installer
  regression tests. All six pass after M3. REG-003 and REG-006 were previously
  skipped due to the bugs fixed in this milestone.

## Upgrade Notes

- **stderr routing change.** `print_message` output now consistently goes to
  stderr. Scripts or automation that captured stdout from the installer to parse
  log messages will need to redirect: `2>&1` or capture stderr explicitly.
- **sudoers file has changed.** The generated sudoers now includes both
  `/bin/systemctl` and `/usr/bin/systemctl` rule variants. Re-run `--prepare`
  and `--install` to update the sudoers file on existing deployments.

## Findings Closed

BASH-006, BASH-016, ARCH-007 (partial), ARCH-008, DEP-007,
REG-001, REG-002, REG-003, REG-004, REG-005, REG-006
