# OraDBA Data Safe Extension v0.11.0

**Release Date:** 2026-02-16

## Overview

Version 0.11.0 introduces standalone usability improvements with a dedicated
version/integrity command and a sourceable shell environment helper.

## Highlights

- **New `ds_version.sh` command** for extension version, metadata, and integrity status.
- **New `datasafe_env.sh` shell helper** to simplify standalone sessions.
- **Idempotent PATH handling** so repeated sourcing does not duplicate entries.
- **Cleaner integrity reporting** with deduplicated file differences.

## Details

### Added

- New script: `bin/ds_version.sh`
  - Shows extension version and metadata from `VERSION` and `.extension`.
  - Reports integrity status using `.extension.checksum`.
  - Supports `-h|--help`, `-V|--version`, and `-c|--changes-only`.
- New script: `bin/datasafe_env.sh`
  - Must be sourced, not executed.
  - Exports `DATASAFE_BASE`.
  - Appends script `bin/` directory to `PATH` once.
  - Adds aliases `ds`, `cdds`, `dshelp`, and `dsversion`.

### Changed

- Standalone docs now use:
  - `source bin/datasafe_env.sh`
  - `source /path/to/odb_datasafe/bin/datasafe_env.sh` in `~/.bash_profile`
- Quick reference includes `ds_version.sh` and `datasafe_env.sh`.
- Extension version bumped to `0.11.0`.

### Fixed

- `ds_version.sh` excludes `.extension.checksum` from additional-file reporting.
- `ds_version.sh` de-duplicates integrity differences when tools emit duplicate lines.

## Testing

```bash
# Validate shell scripts
bash -n bin/ds_version.sh
bash -n bin/datasafe_env.sh

# Verify source-only behavior
bash bin/datasafe_env.sh

# Verify environment setup and aliases
bash -lc 'source ./bin/datasafe_env.sh && echo "$DATASAFE_BASE" && alias dsversion'

# Verify version command
./bin/ds_version.sh --help
./bin/ds_version.sh --changes-only
```

## Breaking Changes

None. Backward compatible for standalone usage; environment helper path moved
from `etc/` to `bin/`.
