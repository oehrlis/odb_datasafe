# OraDBA Data Safe Extension v0.15.2

Release date: 2026-02-19

## Highlights

- Fixed default credential resolution so `DATASAFE_USER` from config is honored.
- Added non-interactive force mode for credential updates.
- Removed plaintext credential payload exposure from debug/error command logging.

## What Changed

### Fixed

- `bin/ds_target_update_credentials.sh`
  - Re-synchronizes `DS_USER`/`DS_SECRET` defaults after `init_config`, ensuring
    values from `datasafe.conf` (for example `DATASAFE_USER=DS_ADMIN`) are used
    when `--ds-user` is not explicitly provided.
  - Added `--force` support to pass OCI update confirmation non-interactively.
  - Uses `--credentials file://...` instead of inline JSON payloads, so command
    logging does not expose plaintext secret values.

### Tests

- `tests/script_ds_target_update_credentials.bats`
  - Added coverage for `--force` option in help/CLI parsing.

## Affected Files

- `bin/ds_target_update_credentials.sh`
- `tests/script_ds_target_update_credentials.bats`
- `CHANGELOG.md`
