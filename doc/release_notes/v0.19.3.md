# Release Notes v0.19.3

## Highlights

Performance patch eliminating in-loop subshell anti-patterns in
`ds_target_details.sh` and `odb_datasafe_help.sh`. On macOS, processing
1400 targets via the payload path saves ~42 s of fork overhead; the
help command drops from ~11 s to under 1 s.

- **Perf**: `ds_target_details.sh` — remove in-loop `tr` and `cut` subshells
- **Perf**: `odb_datasafe_help.sh` — remove in-loop `grep`, `sed`, `cut`, `basename` subshells

---

## Changed

### `bin/ds_target_details.sh` — eliminate in-loop subshells

`collect_target_details()` and `collect_target_details_from_payload()` are
called once per target in a loop. Three patterns were replaced:

- `jq -r '."lifecycle-state" // ""' | tr '[:lower:]' '[:upper:]'` →
  `jq -r '."lifecycle-state" // "" | ascii_upcase'` (removes one `tr` process per target)
- `$(echo "$conn_string" | cut -d: -f1)` / `cut -d: -f2 | cut -d/ -f1` /
  `cut -d/ -f2-` → bash parameter expansion `${%%:*}`, `${#*:}`, `${%%/*}`,
  `${#*/}` (removes 7 process spawns per target when a connection string is present)

### `bin/odb_datasafe_help.sh` — eliminate in-loop subshells

`extract_purpose()` is called once per script in a `for script in *.sh` loop
(~29 scripts). Five patterns were replaced:

- `$(echo "$header" | grep -E ... | head -1 | sed -E ...)` (Formats 1 & 2) →
  `while` + `[[ =~ ]]` + `BASH_REMATCH` (removes 6 process spawns per script)
- `$(echo "$header" | grep -n ... | head -1 | cut -d: -f1)` (Format 3 line
  lookup) → line counter in `while` loop (removes 4 process spawns)
- `$(echo "$line" | sed -E 's/^#\s*//')` → `[[ "$line" =~ ^#[[:space:]]*(.*) ]]`
  + `BASH_REMATCH` (removes 2 process spawns per inner-loop iteration)
- `$(basename "$script")` × 2 → `${script##*/}` (removes 2 process spawns
  per script in the outer loop)
