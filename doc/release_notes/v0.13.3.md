# OraDBA Data Safe Extension v0.13.3

Release date: 2026-02-18

## Highlights

- `ds_target_connect_details.sh` now prints direct `sqlplus` connect strings for each cluster node.

## What Changed

### SQL*Plus Connect Strings in Target Connection Details

Enhanced table output in `bin/ds_target_connect_details.sh` to show executable connect strings directly below `Cluster Nodes` when cluster nodes are available.

The output now includes one line per node in this format:

- `<username>@<hostname>:<listener_port>/<service_name>`

Example:

- `DS_ADMIN@exa117r04db0039:1565/sourci2p.b2x.vwg`
- `DS_ADMIN@exa117r04db0040:1565/sourci2p.b2x.vwg`

This provides a faster handoff from Data Safe target inspection to direct SQL*Plus connectivity checks.
