# Release Notes v0.17.2

## Highlights

This patch release updates default script runtime verbosity for core operational
commands while keeping explicit logging flags unchanged.

## Changed

### Verbose-by-default logging for operational scripts

When no explicit logging option is provided, the following scripts now default
to `INFO` output:

- `bin/ds_target_refresh.sh`
- `bin/ds_target_update_service.sh`
- `bin/ds_target_register.sh`
- `bin/ds_target_update_connector.sh`
- `bin/ds_target_update_tags.sh`
- `bin/ds_target_delete.sh`
- `bin/ds_target_activate.sh`
- `bin/ds_connector_update.sh`
- `bin/ds_target_audit_trail.sh`

### Preserved explicit flag behavior

Explicit logging flags still take precedence over defaults:

- `--quiet` keeps quiet/warn-level output
- `--verbose` sets `INFO`
- `--debug` enables trace/debug output
