# Release Notes v0.17.6

## Highlights

This patch release fixes three bugs in the tag management scripts and adds
configurable, automatic tag value derivation based on compartment name and
target name patterns. Defined tags are now also set at target registration time.

- Fixed broken `--defined-tags` payload (double-wrapped JSON rejected by OCI CLI).
- Fixed missing `--force` that triggered an interactive confirmation prompt,
  causing every update to fail with "invalid input".
- Fixed `DS_ENV_COMP_REGEX` from `datasafe.conf` being silently ignored —
  Environment always resolved to `undef` despite the config being set.
- `ContainerType` is now auto-derived from the target name suffix:
  `_CDBROOT` → `cdbroot`, otherwise `pdb`.
- `Environment` is now derived via a configurable compartment-name regex
  (`DS_ENV_COMP_REGEX` in `datasafe.conf` or `--env-regex` on the CLI).
- `ContainerStage` is auto-derived as `{type}-{env}` (e.g. `pdb-prod`);
  override with `--stage VALUE`.
- `Classification` defaults to `undef`; set with `--class VALUE`.
- `ds_target_register.sh` now sets defined tags at registration time using
  the same derivation logic; skip with `--no-tags`.

## Fixed

### `bin/ds_target_update_tags.sh` — double-wrapped `--defined-tags` JSON

`build_tag_update_json()` produced:

```json
{
  "defined-tags": {
    "DBSec": { "Environment": "undef", … }
  }
}
```

The OCI CLI flag `--defined-tags` already implies the `defined-tags` level.
Wrapping in a second `defined-tags` key caused every update to fail with an
OCI CLI parameter error. Fixed: the function now outputs only the namespace
map:

```json
{
  "DBSec": {
    "Environment": "prod",
    "ContainerStage": "pdb-prod",
    "ContainerType": "pdb",
    "Classification": "undef"
  }
}
```

### `bin/ds_target_update_tags.sh` — missing `--force` triggers interactive prompt

`data-safe target-database update` always prompts for confirmation before
applying changes. Without `--force` in a non-interactive script context, OCI
CLI received `"invalid input"` on every target and exited 1. Fixed by adding
`--force` to the OCI command array in `update_target_tags()`.

### `DS_ENV_COMP_REGEX` from `datasafe.conf` silently ignored

Setting `DS_ENV_COMP_REGEX` in `datasafe.conf` had no effect — `Environment`
always resolved to `undef`. Root cause: the default assignment

```bash
: "${ENV_COMP_REGEX:=${DS_ENV_COMP_REGEX:-}}"
```

runs in the script preamble, **before** `init_config` loads `datasafe.conf`.
At that point `DS_ENV_COMP_REGEX` is empty, so `ENV_COMP_REGEX` gets locked to
`""`. When `init_config` later sets `DS_ENV_COMP_REGEX`, `ENV_COMP_REGEX` is
already set and the `:=` operator does not overwrite it.

**Fixed** in both `ds_target_update_tags.sh` (`validate_inputs()`) and
`ds_target_register.sh` (`derive_tag_values()`): after `init_config` has run,
the function syncs `ENV_COMP_REGEX` from `DS_ENV_COMP_REGEX` if it is still
empty. CLI `--env-regex` sets `ENV_COMP_REGEX` non-empty during argument
parsing, so it always takes precedence over the config value.

## Added

### Configurable tag derivation

| Tag | Source |
| --- | ------ |
| `ContainerType` | Auto-derived: `_CDBROOT` suffix → `cdbroot`, otherwise `pdb`. |
| `Environment` | Capture group 1 of `DS_ENV_COMP_REGEX` / `--env-regex PATTERN` applied to compartment name. `undef` when regex unset or no match. |
| `ContainerStage` | Auto-derived as `{type}-{env}` (e.g. `pdb-prod`, `cdbroot-test`); `undef` when Environment is `undef`. Override: `--stage VALUE`. |
| `Classification` | `undef` by default. Override: `--class VALUE`. |

New CLI flags added to `ds_target_update_tags.sh`:

```text
--env-regex PATTERN   Regex for Environment derivation from compartment name.
                      Capture group 1 is the environment value.
                      Can also be set via DS_ENV_COMP_REGEX in datasafe.conf.
--stage VALUE         Explicit ContainerStage value (overrides auto-derive).
--class VALUE         Explicit Classification value (default: undef).
```

New config key added to `etc/datasafe.conf.example`:

```bash
# DS_ENV_COMP_REGEX='^cmp-.*-([^-]+)-projects$'
```

**Examples:**

```bash
# Set regex once in datasafe.conf, then just run:
# DS_ENV_COMP_REGEX='^cmp-.*-([^-]+)-projects$'
ds_target_update_tags.sh --all --apply

# Or pass inline:
ds_target_update_tags.sh -T exa118r05c19_cdb14a19_PV105VWP --apply \
    --env-regex '^cmp-.*-([^-]+)-projects$'

# Result for a PDB target in cmp-org-unit-prod-projects:
#   Environment:    prod
#   ContainerType:  pdb
#   ContainerStage: pdb-prod
#   Classification: undef

# Result for a CDB root target (_CDBROOT) in the same compartment:
#   Environment:    prod
#   ContainerType:  cdbroot
#   ContainerStage: cdbroot-prod
#   Classification: undef
```

## Added (continued)

### `bin/ds_target_register.sh` — defined tags at registration time

Targets now receive defined tags when registered. The same derivation logic used
by `ds_target_update_tags.sh` is applied so tags are consistent from day one.

| Tag | Source |
| --- | ------ |
| `ContainerType` | Auto-derived: `_CDBROOT` suffix → `cdbroot`, otherwise `pdb`. |
| `Environment` | Capture group 1 of `DS_ENV_COMP_REGEX` / `--env-regex PATTERN` applied to compartment name. `undef` when unset or no match. |
| `ContainerStage` | Auto-derived as `{type}-{env}` (e.g. `pdb-prod`); `undef` when Environment is `undef`. Override: `--stage VALUE`. |
| `Classification` | `undef` by default. Override: `--class VALUE`. |

New CLI flags added to `ds_target_register.sh`:

```text
--no-tags             Skip setting defined tags on the target.
--env-regex PATTERN   Regex for Environment derivation from compartment name.
                      Capture group 1 is the environment value.
                      Can also be set via DS_ENV_COMP_REGEX in datasafe.conf.
--stage VALUE         Explicit ContainerStage value (overrides auto-derive).
--class VALUE         Explicit Classification value (default: undef).
```

**Examples:**

```bash
# Register with tags auto-derived from compartment name pattern
ds_target_register.sh -H db01 --sid cdb01 --pdb APP1PDB \
    -c prod-compartment --connector my-connector --ds-secret <secret> \
    --env-regex '^cmp-.*-([^-]+)-projects$'

# Register without tags
ds_target_register.sh -H db01 --sid cdb01 --pdb APP1PDB \
    -c prod-compartment --connector my-connector --ds-secret <secret> \
    --no-tags
```

The registration plan (shown in dry-run and before submit) now includes the
planned tag values:

```text
[INFO] Tags (DBSec):
[INFO]   Environment:    prod
[INFO]   ContainerType:  pdb
[INFO]   ContainerStage: pdb-prod
[INFO]   Classification: undef
```
