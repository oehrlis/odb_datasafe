# Release Notes - v0.21.0

**Release Date:** 2026-06-28
**Type:** Minor (M1 - Release Gate Restoration)

## Summary

M1 restores the CI/release quality gate that was silently broken: failing tests now
block CI and release. Removes a hardcoded known-credential from the SQL template.
Adds bash 4.0+ runtime enforcement and error protection to two unprotected scripts.

## Changes

### CI and Release Gate

- `make test` now propagates BATS exit code — test failures cause `make test`,
  `make check`, and `make ci` to exit non-zero (was: always exit 0)
- `.github/workflows/ci.yml`: removed `continue-on-error: true` from test step;
  failing tests now block PR merges
- `Makefile`: added `format-check` (shfmt) to `check` and `ci` chains as a hard gate
- `Makefile`: `release` target now requires a passing `check` (lint + format-check + test)
  before bumping the version; a failing codebase can no longer be released

### Security

- `sql/create_ds_admin_user.sql`: removed the hardcoded default password `DS_Admin.2025`
  (publicly committed to GitHub). The default is now `''` and the script fails fast
  with an explicit error when no password is supplied.
- **Action required:** Any deployment where `DS_ADMIN` was created with the default
  password should rotate the account credentials immediately.

### Bash Robustness

- `lib/common.sh`: added bash 4.0+ version guard at source time. Scripts now exit 1
  with a clear message on bash 3.2 (macOS system bash). Install bash 4+ via Homebrew.
- `bin/ds_connector_register_oradba.sh`: added `set -euo pipefail` and
  `setup_error_handling` — OCI CLI failures now propagate correctly
- `bin/ds_connector_update.sh`: same as above

### Documentation

- `doc/milestone-v1.0.0.md`: created v1.0.0 readiness checklist (pass/fail criteria
  for all nine risk-register blockers, regression tests, security, documentation)
- `README.md`, `doc/index.md`: updated "Latest Release" link to v0.20.4

### Code Quality

- `bin/install_datasafe_service.sh`, `bin/uninstall_all_datasafe_services.sh`:
  applied shfmt auto-formatting (spacing around redirections)

## Upgrade Notes

- **bash 4.0+ is now enforced at runtime.** macOS ships bash 3.2; install bash 4+:
  `brew install bash` and ensure `/opt/homebrew/bin` is first in your PATH.
- **DS_Admin.2025 removed from SQL template.** The shell wrapper always passes
  `${DATASAFE_PASS}`, so operational scripts are unaffected. Direct SQL*Plus
  invocations must now pass the password explicitly as parameter 2.
