# OraDBA Data Safe Extension v0.7.1

**Release Date:** 2026-02-11

## Overview

Version 0.7.1 focuses on operational visibility for connector updates and a major
expansion of test coverage. Connector update workflows now include an explicit
version check step, and the test suite gains new edge case and parameter
combination coverage with dedicated documentation.

## Highlights

- **Connector update improvements** in `ds_connector_update.sh` with version
  checks that compare local and online connector versions before updating.
- **Expanded test coverage** with new edge case and parameter combination test
  suites.
- **New testing guide** in `doc/testing.md` plus updated test documentation and
  counts.

## Details

### Connector Update Enhancements

`ds_connector_update.sh` now includes a version check step that:

- Reads local connector version from `setup.py`
- Queries the latest available version from OCI Data Safe
- Compares versions and reports status (up to date, update available, or newer)
- Displays a clear, step-by-step update flow

Example:

```bash
# Run with version check and step-by-step output
bin/ds_connector_update.sh --connector my-connector -c MyCompartment
```

### Test Suite Expansion

New test suites add coverage for edge cases and parameter combinations:

- `tests/edge_case_tests.bats` covers unusual inputs, boundary conditions, and
  error paths
- `tests/integration_param_combinations.bats` validates common option mixes,
  error handling, and basic workflow compatibility

The suite now totals 227+ tests and completes in ~20 seconds.

### Documentation

- Added a comprehensive testing guide in `doc/testing.md`
- Updated `tests/README.md` with new categories, coverage statistics, and
  execution guidance

## Testing

```bash
# Run unit and script tests
make test

# Run the full suite
make test-all
```

## Breaking Changes

None. All changes are backward compatible.
