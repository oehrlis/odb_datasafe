# Release Notes v0.18.1

## Highlights

This patch release fixes two bugs in `ds_connector_create.sh` discovered during
the first real-world install of v0.18.0.

- **Fixed**: bundle download fails immediately after OCI connector creation
  (`NotAuthorizedOrNotFound` 404) — connector still in `CREATING` state.
- **Fixed**: `setup.py install` requires `--connector-port`; added configurable
  `--connector-port PORT` flag (default: `1521`).

---

## Fixed

### Bundle download fails after OCI create (`CREATING` state race)

After `oci data-safe on-prem-connector create` returns successfully, the
connector spends a brief period in `CREATING` state before transitioning to
`INACTIVE`. During that window the bundle download API endpoint
(`generate-on-prem-connector-configuration`) returns HTTP 404
"NotAuthorizedOrNotFound", causing Step 3 to fail immediately.

The workaround was to restart the installation with `--ha-node` (which skips
OCI create and goes straight to bundle download after the connector had settled).

**Fixed**: new `wait_for_connector_not_creating()` function polls OCI every
10 seconds (up to 2 minutes) until the connector's lifecycle state is no longer
`CREATING`, then proceeds with the bundle download. In HA mode the wait is
skipped (the existing connector is already past `CREATING`).

### `setup.py install` requires `--connector-port`

```text
usage: python3 setup.py COMMAND [arguments] install [-h]
       --connector-port <connector-port> [--https-proxy …] …
error: the following arguments are required: --connector-port
```

`run_setup_install()` previously called `setup.py install` without
`--connector-port`. The argument is required by the installer to configure the
port the connector service will listen on.

**Fixed**: added `--connector-port PORT` CLI flag (default: `1521`); the value
is injected into the non-interactive Python wrapper via `CONNECTOR_PORT_INPUT`
and passed as `sys.argv` to `setup.py install`.

## Added

### `--connector-port PORT`

New optional flag for `ds_connector_create.sh`. Default: `1521`.

```bash
# Use default port 1521
ds_connector_create.sh -c my-compartment -N my-connector \
    --connector-home /u01/datasafe/my-connector

# Use a custom port
ds_connector_create.sh -c my-compartment -N my-connector \
    --connector-home /u01/datasafe/my-connector \
    --connector-port 1522
```

The port is shown in the plan log:

```text
[INFO] Connector Creation Plan:
[INFO]   Display name:    my-connector
[INFO]   Compartment:     my-compartment (ocid1.compartment…)
[INFO]   Connector home:  /u01/datasafe/my-connector
[INFO]   Connector port:  1521
```
