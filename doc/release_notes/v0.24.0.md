# Release Notes - v0.24.0

**Release Date:** 2026-06-28
**Type:** Minor (M4 - Test Coverage & Robustness)

## Summary

M4 is the largest milestone to date, driven by five parallel workstreams. It adds
REG-007..REG-012 regression tests, a new `lib_ssh_helpers.bats` suite (17 tests),
bounded parallelism for bulk async operations (PERF-012), input-injection defenses
(BASH-007/008, ORA-015), O(N) call elimination in `ds_target_connector_summary.sh`
(PERF-001), strict-mode bootstrap hardening across all 21 scripts (BASH-001), and a
sweep of security and robustness fixes in `lib/oci_helpers.sh`. Total test count:
349 (up from 308 after M3).

## Changes

### Regression Tests (REG-007..REG-012)

- **REG-007** (`lib_oci_helpers.bats`): `oci_exec` stderr isolation. Mock OCI emits
  Python FutureWarning to stderr and valid JSON to stdout; asserts output contains
  only JSON.
- **REG-008** (`lib_oci_helpers.bats`): `ds_is_updatable_lifecycle_state` coverage -
  DELETED state rejected, ACTIVE and NEEDS_ATTENTION states accepted.
- **REG-009** (`script_ds_target_update_service.bats`): PUT semantics - verifies
  `target-database get` is called before `target-database update` in `--apply` mode.
- **REG-010** (`script_ds_target_activate.bats`): Multi-target partial failure - one
  target fails, second succeeds; script processes both and exits 10 (partial-failure
  code) rather than aborting.
- **REG-011/REG-012** (`lib_common.bats`): `normalize_secret_value` - file-path input
  returns file contents; literal string input returned unchanged.

### New Test File: tests/lib_ssh_helpers.bats

17 tests covering all four functions in `lib/ssh_helpers.sh`:
`ssh_require_tools`, `ssh_exec`, `ssh_scp_to`, `ssh_check`. Includes default variable
value assertions and expected-failure tests to non-routable hosts.

### Performance (PERF-001, PERF-002, PERF-003, PERF-004, PERF-012)

- **PERF-001**: `bin/ds_target_connector_summary.sh` - `ENRICH_MISSING` now defaults
  to `false`. Eliminates O(N) per-target OCI GET calls on every invocation. Use new
  `--enrich` flag to opt in.
- **PERF-002**: `ds_resolve_target_name()` - accepts optional pre-resolved name as
  second argument, skipping OCI GET when caller has already resolved it.
- **PERF-003**: `oci_resolve_compartment_ocid()` - in-memory associative-array cache
  (`_COMP_OCID_CACHE`) eliminates repeated compartment resolutions within a single
  script invocation.
- **PERF-004**: `ds_is_cdb_root_target()` - accepts optional pre-fetched target JSON
  as second argument, skipping per-target OCI GET when data is already available.
- **PERF-012**: `bin/ds_target_refresh.sh` and `bin/ds_target_activate.sh` - `--mode
  async` now runs up to `MAX_PARALLEL` (default 4) background jobs concurrently.
  New `--max-parallel N` CLI flag; `MAX_PARALLEL` env var also supported. Serial mode
  (`--mode sync`, default) behavior unchanged.

### Input Safety (BASH-007, BASH-008, ORA-015, DEP-006)

- **BASH-008**: `bin/ds_target_register.sh` - all `jq` filter strings that previously
  interpolated `$VAR` directly now use `jq --arg` to prevent injection via values
  containing `"` or `\`.
- **BASH-007**: `bin/ds_target_register.sh` - compartment name validated against a
  safe-character allowlist before embedding in OCI CLI `--query` JMESPath expressions.
- **ORA-015**: `bin/ds_database_prereqs.sh` - `--ds-user` and `--pdb` values validated
  against Oracle identifier rules (letter start, `A-Za-z0-9_$#`, max 128 chars) before
  embedding in SYSDBA SQL.
- **DEP-006**: `bin/ds_database_prereqs.sh` - `ORACLE_HOME` validated (set, directory
  exists, `sqlplus` executable) before attempting SQL execution.

### Strict-Mode Bootstrap (BASH-001)

- `setup_error_handling` moved to before `init_config` in all 21 scripts that had the
  call order reversed. Error traps are now active during config loading.
- `setup_error_handling` added to `bin/ds_target_move.sh`, `bin/ds_version.sh`, and
  `bin/template.sh` which had `init_config` but no error handling setup.

### Bash Robustness (ARCH-005, ARCH-011, BASH-004, BASH-013, DEP-005, SEC-010)

- **ARCH-005/BASH-014**: `ds_refresh_target()` in `lib/oci_helpers.sh` - replaced
  `"${cmd[@]}" 2>&1` with explicit stderr temp-file capture to prevent Python
  FutureWarnings from corrupting JSON stdout.
- **ARCH-011**: `resolve_compartment_to_vars()` and `resolve_target_to_vars()` in
  `lib/oci_helpers.sh` - replaced `eval "${prefix}_X=..."` with
  `printf -v "${prefix}_X" '%s' "$value"` to eliminate code injection risk.
- **BASH-004**: `bin/ds_target_move.sh` - three `((count++))` expressions replaced
  with `count=$((count + 1))` to avoid `set -e` exit when count is 0.
- **BASH-013**: `generate_bundle_key()` in `lib/oci_helpers.sh` - unbounded `while
  true` loop bounded to 50 iterations; returns error on exhaustion.
- **DEP-005**: `bin/ds_target_move.sh` - added `require_oci_cli` call in validation.
- **SEC-010**: `_oci_redact_cmd()` in `lib/oci_helpers.sh` - now masks `--credentials`,
  `--secret`, and `--auth-token` in addition to `--password`.

### Test Quality (TEST-012)

- `tests/lib_oci_helpers.bats`: 6 zero-signal `[ "$status" -eq 0 ] || [ "$status" -eq 1 ]`
  assertions replaced with meaningful checks. Tests 4 and 6 rewritten to use
  `bash -c` subprocess to correctly scope `declare -A` associative arrays.
- `tests/uninstall_all_datasafe_services.bats`: 3 zero-signal assertions fixed.

## Upgrade Notes

- **ENRICH_MISSING behavior change.** `bin/ds_target_connector_summary.sh` no longer
  enriches missing-connector entries by default. If your workflow relied on per-target
  detail for unlinked targets, add `--enrich` to your invocation.
- **setup_error_handling before init_config.** Scripts now install error traps before
  sourcing config files. Config syntax errors that were previously silent will now abort
  with a clear error message and line number.
- **ds_target_connector_summary.sh --no-enrich retained.** Existing scripts using
  `--no-enrich` continue to work (it is now a no-op since false is the default).
