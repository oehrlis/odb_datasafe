## Changed

### `Makefile` â€” OraDBA standard preamble and release workflow

Preamble hardening:

- `SHELL := /bin/bash` (was `/usr/bin/env bash`)
- `.SHELLFLAGS := -eu -o pipefail -c` enforces `set -euo pipefail` for every recipe
- `MAKEFLAGS += --no-builtin-rules` removes implicit rule noise
- `PATH="$(PATH)"` prefix on all `command -v` tool-detection calls

Release workflow additions:

- `release` target orchestrates `version-bump-patch` â†’ commit â†’ tag in one step
- `tag` target guards against dirty tree, uncommitted VERSION, and existing tags
- `check-version` target validates VERSION format; wired as a `lint` dependency
- `version-bump-*` now commits `VERSION` + `.extension` together and uses
  `perl -pi -e` instead of `sed -i` for macOS/Linux compatibility

Verbosity control: `V ?= 0` / `Q := @` pattern; `$(Q)` prefix on recipe lines.

`TEST_TIMEOUT` raised from `800` to `1800` seconds to accommodate slow CI environments.

### `.github/workflows/ci.yml` and `release.yml`

- OraDBA file headers added
- `ci.yml`: `check-version` step inserted before `lint-shell`
- `release.yml`: validate-version-format step added; stray `make install` removed
