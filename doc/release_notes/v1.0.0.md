# Release Notes - v1.0.0

**Release Date:** 2026-06-28
**Type:** Major (M5 - Documentation & Polish; library domain refactor)

## Summary

v1.0.0 completes the five-milestone roadmap from v0.20.4. The headline change is
ARCH-013: all 34 `ds_*` domain functions migrate from `lib/oci_helpers.sh` (2310 lines)
into a new `lib/ds_lib.sh` (1596 lines), leaving `lib/oci_helpers.sh` as a focused
726-line OCI primitives module. Accompanying this are portability fixes (DEP-003 —
BSD-safe `grep -oE`), correctness fixes (BASH-005, BASH-018, PERF-007), documentation
accuracy across all user-facing docs, three permanently skipped tests removed (TEST-016),
and release-process cleanup (REL-008, REL-009).

Total test count: **346** (349 after M4, minus 3 permanently skipped tests removed).

## Changes

### Library Refactor — ARCH-013

`lib/oci_helpers.sh` previously hosted 34 `ds_*` Data Safe domain functions
alongside the OCI primitives, growing to 2310 lines. The domain layer is now
isolated in `lib/ds_lib.sh`:

```text
lib/oci_helpers.sh  2310 → 726 lines  (OCI primitives only)
lib/ds_lib.sh         39 → 1596 lines (all ds_* domain functions)
```

`lib/ds_lib.sh` self-sources `common.sh`, `oci_helpers.sh`, and `ssh_helpers.sh`
with a double-source guard (`DS_LIB_SH_LOADED`). All bin scripts that previously
sourced `oci_helpers.sh` for `ds_*` functions source `ds_lib.sh` instead.
`tests/lib_oci_helpers.bats` and `tests/bash42_compatibility.bats` updated
to source `ds_lib.sh` where needed.

Related deduplication:

- **ARCH-003**: duplicate `is_ocid` function removed from `lib/oci_helpers.sh`
- **ARCH-004**: stale `_ds_cache_mtime` function deleted; callers use `_ds_file_mtime`

### Portability (DEP-003)

`bin/ds_connector_update.sh` used `grep -oP` (PCRE) for version string extraction.
PCRE is not available in BSD `grep` (macOS). Replaced with `grep -oE '[0-9]+\.[0-9]+(\.[0-9]+)?'`.

### Bash Correctness (BASH-005, BASH-018)

- `lib/common.sh` line 25: `export LC_ALL=C` added for deterministic locale
  handling across all invocations.
- `lib/common.sh` stack-trace: `((frame++))` replaced with `((frame++)) || true`
  to avoid `set -e` exit when the frame counter starts at 0.

### Performance (PERF-007)

`bin/ds_target_list.sh` state aggregation: `sort | uniq -c | sort -rn` pipeline
replaced with a `jq` group-by expression. Eliminates the external `sort`/`uniq`
subprocesses and produces correct counts when target state values contain spaces.

### Test Quality (TEST-016)

Three `@test` blocks in `tests/edge_case_tests.bats` that had been permanently
skipped (via unconditional `skip`) with no path to removal were deleted. This
corrects the reported test count from 349 to **346**.

### Release Engineering (REL-008, REL-009)

- `.github/workflows/release.yml`: redundant `make test` step removed. The
  preceding CI workflow already runs `make ci` (which includes tests); running
  `make test` again in the release workflow was pure overhead.
- `scripts/build.sh`: timestamp removed from `.extension.checksum`. The checksum
  file now contains only content hashes, making builds reproducible.

## Upgrade Notes

- **lib/ds_lib.sh is now required.** Any external script or tool that sources
  `lib/oci_helpers.sh` directly and calls `ds_*` functions must be updated to
  source `lib/ds_lib.sh` instead (or source it in addition). All bundled `bin/`
  scripts are updated.
- **grep -oP removed.** If any local override of `bin/ds_connector_update.sh`
  relies on PCRE syntax, update to ERE syntax.
- **Test count is 346**, not 349. The three removed tests in
  `tests/edge_case_tests.bats` were permanently skipped and are gone from the
  suite.
